    03 · Beyond the Browser
    Cortex lives outside the tab

    🔌 Chrome Extension Manifest V3

    • Timer runs in background service worker — popup can be closed
    • Site blocking via declarativeNetRequest — HTTP level, unbypassable
    • 5 tabs: Timer · Blocker · Social · History · Settings
    • Same JWT as web app — one login, one account
    • Sessions sync → same leaderboard as web

    🐧 GNOME Extension Tamatem · tamatem@frey.dev

    • Persistent Pomodoro timer in the Linux system top bar
    • OS-level site blocking — modifies hosts/DNS, not browser-specific
    • Written in GJS, uses GNOME Shell APIs (v48–50)
    • Syncs sessions to same Cortex backend
    • For CS students on Ubuntu/Fedora — no browser required
    ✅ Three clients · One login · Same data · Same leaderboard
    04 · Architecture
    Three clients. One backend. One rule.
    • ⚠️The Rule: No client ever calls Supabase directly — always through Express
    • 🔑Why: service role key bypasses ALL security — lives only on Railway
    • 🏗️7 domains: Auth · Notes · Daily · Data · AI · Admin · Workspace
    • 🔁All 3 clients use same JWT → same Express API → same response
    05 · Backend Pattern
    Every domain — same three layers
    • 🎮Controller — HTTP in/out, parse & validate, format response. Zero logic.
    • ⚙️Service — business logic, orchestration, cross-service calls. No DB.
    • 🗃️Repository — ALL Supabase queries. Nothing else. Ever.
    • ✅Same structure enforced across all 7 domains
    // Strict rule — no exceptions
    Controller → Service → Repository
    No Supabase calls outside Repository

    07 · Technology Decisions
    Nothing was a default
    Plate.js
    vs Quill, TipTap
    Produces structured JSON — backend walks the tree for AI text extraction. Quill produces HTML → brittle to parse.
    Gemini
    vs OpenAI, Anthropic
    Best Arabic quality · 100–400× cheaper than GPT-4 · only provider with both quality chat AND embedding in one API
    Supabase
    vs Firebase, raw PG
    RLS + pgvector + Auth bundled. Firebase is NoSQL — incompatible with our 44-migration relational schema.
    Next.js 15
    vs Vite SPA
    Server Components render catalog on server — instant, no JS loading state. Admin guards run server-side.
    Separate Express
    vs Next.js routes
    Service role key must be isolated from Vercel. 3 clients need one shared API — not a frontend implementation detail.
